    Docker and Containers

    Containers Introduction

    ‚ùå Problems with Traditional Servers

    • One application per dedicated server
    • Physical servers required
    • Operating system licenses may be needed
    • Underutilization of server resources
    • Many other challenges...

    ‚úÖ Solution: Virtualization

    • Use of Hypervisors to create virtual machines:
    • KVM
    • Xen
    • VMware
    • Oracle VirtualBox
    • ...

    üéØ The Main Value for Customers

    • It‚Äôs the application that matters!
    • The operating system itself does not add business value.

    ‚ùå Problems with Virtual Machines (VMs)

    • Operating system does not contribute directly to business value.
    • The OS consumes a significant amount of VM resources.

    üì¶ What Are Containers?

    • Containers are lighter than virtual machines.
    • They consume less CPU, RAM, and disk space.
    • Containers are an OS feature in UNIX-like systems.

    üñ•Ô∏è Containers in Linux

    • Use fewer resources compared to VMs.
    • Improve efficiency by isolating applications.

    ‚öôÔ∏è How Containers Work

    • Sometimes, applications need different versions of the same library.
    • Containers solve dependency conflicts by providing isolated environments.

    üõ†Ô∏è Kernel Features for Containers

    • Namespaces (process isolation):
    • PID ‚Äì process isolation
    • MNT ‚Äì file system isolation
    • NET ‚Äì network isolation
    • USER ‚Äì user privilege isolation
    • Control Groups (cgroups) ‚Äì group resources and set limits.
    • Capabilities ‚Äì control privileges.

    üö¢ Containerization

    • Achieved through platforms like Docker and Kubernetes.
    • Uses technologies like Linux Containers (LXC) and containerd.

    üîë Key Features of Containerization

    1. Isolation
    2. Portability
    3. Efficiency
    4. Reproducibility
    5. Scalability

    Docker Introduction

    üê≥ What is Docker?

    • A set of PaaS (Platform as a Service) products.
    • Uses OS-level virtualization to deliver software in containers.

    ü§î Why Use Docker?

    • Isolation ‚Äì Keeps applications separate.
    • Lightweight ‚Äì Uses fewer resources than VMs.
    • Simplicity ‚Äì Easy to set up and use.
    • Workflow ‚Äì Streamlines development and deployment.
    • Community ‚Äì Large ecosystem of support.

    üèóÔ∏è Docker Structure

    üîß Docker Engine

    • Client-server application that builds and runs containers.
    • REST API enables communication between Docker Client and Docker Daemon.

    üìñ Docker Terminology

    • Docker Container ‚Äì An encapsulated application with all dependencies, sharing the host OS kernel.
    • Docker Image ‚Äì A template for creating containers, containing all necessary configurations.
    • Registry ‚Äì A repository storing Docker images (e.g., Docker Hub).

    üèóÔ∏è Installing Docker

    üîó Official Documentation: - Docker Installation for Ubuntu - Digital Ocean Guide

    ‚ö° Docker Engine Commands

    üìú Docker Images

    • Built from a series of layers.
    • Created using Dockerfile.
    • Not a filesystem, but a compressed archive (tar file).

    üìù Dockerfile

    • Automates image creation using a script-like format.
    • Extends from a Base Image.
    • Example: 
      # Base image
FROM ubuntu:latest
# Install dependencies
RUN apt-get update && apt-get install -y python3
# Set working directory
WORKDIR /app

    üõ†Ô∏è Docker Instructions

    • FROM ‚Äì Defines the base image.
    • RUN ‚Äì Executes commands inside the container.
    • CMD ‚Äì Specifies the default command.
    • COPY / ADD ‚Äì Copies files from the host to the container.
    • EXPOSE ‚Äì Specifies the port the container listens on.
    • ENTRYPOINT ‚Äì Defines the main command for the container.

    üèóÔ∏è Creating and Running Docker Containers

    üîç Viewing Images & Containers

    docker images    # List available images
docker ps        # List running containers
docker ps -a     # List all containers (including stopped ones)

    üöÄ Running a Container

    docker run --name my-container -dit ubuntu
    - -d ‚Äì Runs in detached mode (background). - -i ‚Äì Interactive mode. - -t ‚Äì Allocates a terminal.

    üõë Stopping and Restarting Containers

    docker stop <container_id>
docker start <container_id>

    üîÑ Copying Files into a Container

    docker cp my_file.txt <container_id>:/app/

    üõ†Ô∏è Executing Commands Inside a Running Container

    docker exec -it <container_id> bash

    üèóÔ∏è Building an Image from a Dockerfile

    docker build -t my-image .

    üèõÔ∏è Case Studies: Docker in Ubuntu

    üèóÔ∏è Setting Up Docker on Ubuntu

    sudo apt update
sudo apt install apt-transport-https ca-certificates curl software-properties-common
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install docker-ce

    üèóÔ∏è Downloading Images

    docker pull ubuntu
docker pull openjdk:8-jdk

    üèóÔ∏è Creating a Container

    docker run --name webserver -dit -p 8080:80 httpd
    - Exposes port 80 inside the container and maps it to 8080 on the host.

    üèóÔ∏è Verifying the Application

    curl http://localhost:8080

    üóëÔ∏è Removing Containers and Images

    docker rm <container_id>
docker rmi <image_id>

    üìú Writing a Dockerfile (Example)

    Cloning a Project from GitHub

    git clone https://github.com/mentorchita/my_yourname_site.git
cd my_yourname_site
touch Dockerfile

    Basic Dockerfile Example

    # Use an official Apache image
FROM httpd:2.4
# Copy website files into the container
COPY ./my_yourname_site /usr/local/apache2/htdocs/

    Building and Running the Image

    docker build -t my-apache2 .
docker run -dit --name my-running-app -p 8080:80 my-apache2

    üéØ Conclusion

    • Docker simplifies application deployment by packaging everything into containers.
    • Lightweight and efficient, compared to virtual machines.
    • Easily portable across different environments.
    • Great for CI/CD workflows and scalable microservices.
